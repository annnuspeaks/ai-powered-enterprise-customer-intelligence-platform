# 2.2.3 API Design

## API Overview

The backend exposes RESTful APIs for data upload, prediction, dashboard analytics, and report generation. All responses are returned in JSON format.

---

## Base URL

```
/api/v1
```

---

## Data Management APIs

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | /upload | Upload customer dataset |
| GET | /customers | Retrieve customer data |

---

## Prediction APIs

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | /predict/churn | Predict customer churn |
| POST | /predict/segment | Predict customer segment |
| POST | /predict/clv | Predict customer lifetime value |
| POST | /predict/propensity | Predict purchase propensity |

---

## Dashboard APIs

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | /dashboard/kpi | Retrieve KPI metrics |
| GET | /dashboard/charts | Retrieve dashboard chart data |
| GET | /dashboard/summary | Retrieve business summary |

---

## Report APIs

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | /reports/export | Export prediction results |

---

## Future APIs

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | /auth/login | User authentication |
| POST | /auth/register | User registration |
| GET | /history | Prediction history |

---

## Response Format

### Success Response

```json
{
  "status": "success",
  "message": "Request processed successfully.",
  "data": {}
}
```

### Error Response

```json
{
  "status": "error",
  "message": "An error occurred."
}
```

# 2.2.2 Database Design

## Database Overview

The platform uses a relational database to store processed customer information, prediction results, and application data.

Database Technology:

- SQLite (Minor Project)
- PostgreSQL (Major Project)

---

# Database Entities

The application consists of the following primary entities:

- Customers
- Predictions
- Prediction History (Future)
- Users (Future)

---

# Entity Description

## Customers

Stores processed customer information used for Machine Learning predictions.

Attributes:

- customer_id
- customer_unique_id
- customer_city
- customer_state
- recency
- frequency
- monetary
- average_order_value
- total_orders
- total_spent

---

## Predictions

Stores prediction results generated by Machine Learning models.

Attributes:

- prediction_id
- customer_id
- churn_prediction
- customer_segment
- predicted_clv
- purchase_propensity
- prediction_date

---

## Prediction History (Future)

Stores historical prediction records for tracking and comparison.

Attributes:

- history_id
- customer_id
- prediction_id
- created_at

---

## Users (Future)

Stores administrator information.

Attributes:

- user_id
- name
- email
- password_hash
- role

---

# Entity Relationships

Customers

↓

Predictions

↓

Prediction History

Users manage customer data and predictions.

---

# Initial Database Schema

Customers (1)

↓

Predictions (Many)

↓

Prediction History (Many)

---

# Database Design Principles

- Relational Structure
- Normalized Schema
- Modular Design
- Scalable Architecture
- Easy Migration to PostgreSQL
- Future Authentication Support

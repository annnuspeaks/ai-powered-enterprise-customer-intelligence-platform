# 2.1.2 Requirements Specification

## Functional Requirements

The system shall:

- Upload customer datasets (CSV format).
- Validate uploaded datasets.
- Perform automated data preprocessing.
- Generate customer churn predictions.
- Perform customer segmentation.
- Predict Customer Lifetime Value (CLV).
- Predict purchase propensity.
- Display business insights through an interactive dashboard.
- Explain predictions using Explainable AI.
- Export prediction results and reports.
- Provide REST APIs for frontend integration.
- Support deployment as a live web application.

---

## Non-Functional Requirements

The system should:

- Be responsive and user-friendly.
- Provide accurate and reliable predictions.
- Maintain modular and scalable architecture.
- Process data efficiently.
- Support future feature expansion.
- Be deployable on cloud platforms.
- Follow secure coding practices.
- Ensure maintainable and well-documented code.

---

## User Roles

### Administrator

- Manage the platform.
- Upload datasets.
- View analytics.
- Generate predictions.
- Export reports.

### Business User

- Analyze customer insights.
- View dashboards.
- Download reports.

---

## Assumptions

- Input datasets are provided in CSV format.
- Users have basic knowledge of customer analytics.
- Internet connectivity is available for the deployed application.

---

## Constraints

- Initial development targets small to medium-sized datasets.
- Cloud deployment will use free or minimal-cost services.
- Authentication will be implemented during the Major Project phase.

---

## Acceptance Criteria

The project will be considered complete when:

- All ML modules produce predictions successfully.
- Dashboard displays business insights correctly.
- REST APIs function as expected.
- The application is deployed successfully.
- Documentation is complete.
